Rules and Guidelines

Team Formation : Teams of 3-6 members are allowed, with cross-branch, cross-year, and interdisciplinary team formations encouraged.

Project Alignment : Projects must align with the given themes or problem statements and follow event guidelines.

Hackathon Schedule : The hackathon begins on 16th November at 8:00 p.m. when problem statements are released. Final project submission is due by 3:30 p.m. on 18th November. No edits allowed after this deadline.

Original Work : All code must be original and developed during the hackathon. Exact replicas of any already existing websites are strictly prohibited, whereas solutions with changes and additional features are encouraged (open-source libraries, frameworks, and GPT tools are allowed).

Deployment : Projects must be locally deployed and should not rely on direct-use software or pre-built platforms for primary functionality. Direct deployment on tools like website builders or no-code platforms is prohibited. All code should be developed and run independently to demonstrate original work.

Independence : Each team must work independently; cross-team collaboration is prohibited. Mentors and organizers are available for guidance.

Project Presentation : At the hackathon‚Äôs end, each team will present its project, showcasing functionality, features, and innovations.

Judging Criteria : Judging will consider innovation, technical implementation, user experience, scalability, and theme relevance. The judges' decision is final.

Project Ownership : Participants retain full ownership of their projects, though selected projects may be showcased by organizers.

Submission Deadline : Teams must submit all code and documentation by the deadline. Late submissions may result in disqualification.

Post-Event Sharing : Post-event, participants may share projects with the wider community (e.g., on GitHub).

Code of Conduct : All participants must adhere to a code of conduct promoting inclusivity, respect, and professionalism.

Media Use Consent : By participating, you consent to media use of photos, videos, or project details by organizers for promotional purposes.

Liability : Organizers hold no liability for damages, losses, or injuries. Participants are responsible for their own safety and equipment.

Dispute Resolution : Any disputes will be resolved through the organizers‚Äô final decisions. Rule changes may occur and will be promptly communicated to all.
